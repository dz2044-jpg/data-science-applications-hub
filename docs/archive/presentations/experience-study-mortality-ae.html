The Core Purpose: Config-Backed Mortality Exploration

Experience Study Mortality A/E turns one saved mortality configuration into a flexible analytical surface for repeated hypothesis testing.

  • A saved mortality configuration reloads the dataset, mappings, and analysis context without remapping files.

  • The x-axis can be numeric, date, categorical, or A x B, so analysts can test real business hypotheses instead of canned cuts.

  • A/E by Count and A/E by Amount sit side-by-side with 95% confidence intervals so signal and credibility stay visible together.

  • Optional split overlays turn one run into a comparison surface.

Slide 02

The Analytical Edge: Diagnostic Insights That Drill Back

The module does not start from a blank canvas. It starts from machine-ranked segments that can be drilled straight back into the mortality controls.

  • DuckDB-ranked 1D and 2D segments are generated from the saved dataset configuration before manual slicing starts.

  • Insights surface sample, actual, expected, variance, and A/E for both Count and Amount in one ranked view.

  • One click drills the winning segment back into the mortality controls.

  • The workflow starts from the strongest signal instead of a blank canvas.

Slide 03

From Signal to Explanation: Decomposition In The Same Workflow

The module keeps the follow-up analysis inside the same run, so the team can move from an elevated segment to an explanation without exporting work.

  • Once a segment is identified, the module keeps the follow-up inside the same config-backed workflow.

  • Treemap views show A/E intensity against policy count and face amount so concentration and severity can be read together.

  • Claim Analysis by Cause of Death adds claim tables and COLA M1 stacked bars to explain what is driving elevated experience.

  • The module moves from detection to decomposition without exporting work to another spreadsheet.
